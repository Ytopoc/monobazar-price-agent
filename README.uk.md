# Monobazar Pricing Agent

Сервіс автоматичної рекомендації ціни для оголошень про продаж б/в товарів на маркетплейсі Monobazari. На основі фото та опису від продавця повертає три цінові стратегії (швидкий продаж / збалансована / максимум) разом з очікуваним часом продажу і коротким поясненням.

## Як це працює

Ціноутворення відбувається у дві фази:

**Фаза 1 - зрозуміти товар.** Сервіс визначає категорію, проганяє опис через regex-екстрактори і паралельно надсилає фото у vision-API для оцінки стану та витягування відсутніх атрибутів (бренд, модель, дефекти тощо). Якщо якийсь критичний атрибут досі відсутній - продавця запитують 1–3 короткі уточнювальні питання.

**Фаза 2 - назначити ціну.** Три джерела сигналу запускаються паралельно:
- **FAISS** - топ-N схожих оголошень з векторного індексу окремої категорії (ембединги: `intfloat/multilingual-e5-large`).
- **OLX** - актуальні ціни конкурентів через офіційний OLX API (з fallback через DuckDuckGo).
- **LightGBM** - три квантильні моделі (q25 / q50 / q75) плюс модель днів-до-продажу, навчені на історичних SOLD-оголошеннях. Ознаки: атрибути товару плюс агрегована статистика по сусідах FAISS.

Прогнози моделі далі коригуються за станом, здоров'ям батареї (для смартфонів) і коефіцієнтом торгу для категорії. Фінальні описи стратегій і текст оголошення генеруються LLM-API; якщо LLM недоступний - вмикається детермінований шаблонний fallback.

## Підтримувані категорії

| ID | Категорія |
|----|-----------|
| 4 | Смартфони Apple |
| 512 | Кросівки |
| 743 | Конструктори (LEGO тощо) |
| 795 | Книги |
| 1261 | Шини, диски, колеса |
| 1320 | Стільці та крісла |
| 1677 | Колекційні фігурки |

Кожна категорія має власний regex-екстрактор атрибутів, власний FAISS-індекс і власні навчені LightGBM-моделі.

## Швидкий старт

```bash
# 1. Встановити залежності
pip install -r requirements.txt

# 2. Налаштувати креди (зберігаються ПОЗА деревом проєкту)
mkdir -p ~/.monobazar-pricing-agent
cp .env.example ~/.monobazar-pricing-agent/.env
# відредагуйте ~/.monobazar-pricing-agent/.env і заповніть ANTHROPIC_API_KEY (і опційно SERPAPI_KEY)

# 3. Зібрати FAISS-індекси і навчити цінові моделі
python scripts/build_index.py

# 4. Запустити API + UI
uvicorn api.app:app --reload
# відкрити http://localhost:8000
```

> **Шлях до кредів.** `config/settings.py` шукає `.env` спочатку за `~/.monobazar-pricing-agent/.env`, потім у корені проєкту, потім за шляхом у змінній середовища `MONOBAZAR_ENV_FILE`. Рекомендоване розташування - у домашній директорії, бо дерево проєкту ніколи не містить справжніх ключів і їх неможливо випадково закомітити.

## API

| Метод | Шлях | Призначення |
|-------|------|-------------|
| GET | `/api/v1/health` | Статус сервісу; кількість завантажених категорій і моделей |
| POST | `/api/v1/price` | Повне ціноутворення (Фаза 1 + Фаза 2) |
| POST | `/api/v1/price/clarify` | Продовження після відповідей продавця на уточнювальні питання |
| POST | `/api/v1/price/quick` | Швидка оцінка без OLX і LLM (тільки локальні моделі) |
| POST | `/api/v1/pre-analyze` | Тільки Фаза 1: атрибути + уточнювальні питання |
| POST | `/api/v1/analyze-photos` | Глибокий аналіз фото (стан, дефекти, бренд, модель) |

Тіло запиту `POST /api/v1/price`:
```json
{
  "description": "iPhone 13 Pro 256GB синій, батарея 92%",
  "photos": ["<base64-jpeg>", "..."],
  "category_id": 4
}
```

Якщо `category_id` не вказано, категорія визначається з ключових слів опису.

## Структура проєкту

```
monobazar-pricing-agent/
├── api/             FastAPI: app.py, routes.py, Pydantic-схеми
├── agent/           Фаза 1 (ClarificationAgent), Фаза 2 (PricingAgent), шаблони промптів
├── config/          Налаштування (.env-завантажувач) + конфіги категорій (атрибути, regex, ключові слова)
├── features/        Regex-екстрактори по категоріях + обчислення і ранжування ознак для порівняння
├── search/          E5-ембединги, FAISS-індекси, OLX-пошук
├── ml/              LightGBM-тренер + предиктор (квантильна регресія)
├── scripts/         build_index.py - збірка індексів + тренування; evaluate.py - оцінка якості
├── tests/           pytest-набори для extractor / comparable / olx_search
├── ui/              index.html - mobile-first веб-UI
└── data/
    ├── raw/         Вихідні CSV (оголошення, фото)
    ├── indices/     Збережені FAISS-індекси (*.faiss + *_meta.pkl)
    ├── models/      Натреновані LightGBM-моделі (*.pkl) + category_stats.pkl
    └── evaluation/  Результати оцінки (метрики, графіки)
```

## Тренування та оцінка

```bash
# Зібрати індекси і навчити моделі для всіх категорій
python scripts/build_index.py

# Тільки індекси (без тренування)
python scripts/build_index.py --skip-train

# Тільки тренування (індекси вже існують)
python scripts/build_index.py --skip-faiss

# Одна категорія
python scripts/build_index.py --categories 4

# Оцінка на test-сплиті (за замовч.: SOLD-оголошення створені після 2026-03-01)
python scripts/evaluate.py
```

Метрики, що відстежуються по категоріях: MAPE, MAE, медіана APE, покриття інтервалу [q25, q75], середня впевненість моделі. Результати записуються у `data/evaluation/`.

## Тести

```bash
# Юніт-тести regex-екстрактора
python tests/test_extractor.py

# Логіка comparable-ознак і ранжування
python tests/test_comparable.py

# OLX-пошук (потребує мережу)
pytest tests/test_olx_search.py
```

## Технологічний стек

- **API:** FastAPI + Uvicorn
- **ML:** LightGBM (квантильна регресія), scikit-learn
- **Пошук:** FAISS (IndexFlatIP, cosine), sentence-transformers (`intfloat/multilingual-e5-large`)
- **Скрейпінг:** офіційний OLX API + DDGS fallback
- **LLM:** Anthropic API (vision для аналізу фото, текст для пояснень і тексту оголошення)
- **UI:** ванільний HTML/CSS/JS (без фреймворків), mobile-first

## Безпека

- `.env` у `.gitignore` і **не** зберігається у дереві проєкту за замовчуванням.
- `.env.example` містить тільки placeholder-и.
- Рекомендоване розташування кредів: `~/.monobazar-pricing-agent/.env` - повністю поза репозиторієм, тому їх неможливо випадково додати у staging.
- Якщо колись вставили справжній ключ у дерево проєкту - негайно перегенеруйте на https://console.anthropic.com/settings/keys.
