# Monobazar Pricing Agent

Automatic price-recommendation service for second-hand listings on the Monobazari marketplace. Given seller-supplied photos and a description, it returns three pricing strategies (fast sale / balanced / maximum) along with an estimated time-to-sell and a short explanation.

## How it works

Pricing runs in two phases:

**Phase 1 - understand the item.** The service detects the category, runs the description through regex extractors, and in parallel sends the photos to a vision API to assess condition and pull missing attributes (brand, model, defects, etc.). If any critical attribute is still missing, the seller is asked 1–3 short clarifying questions.

**Phase 2 - price it.** Three signal sources run in parallel:
- **FAISS** - top-N similar listings from a per-category vector index (embeddings: `intfloat/multilingual-e5-large`).
- **OLX** - live competitor prices via the official OLX API (with a DuckDuckGo fallback).
- **LightGBM** - three quantile models (q25 / q50 / q75) plus a days-to-sell model, trained on historical SOLD listings. Features: item attributes plus aggregate statistics over the FAISS neighbours.

Model predictions are then adjusted for condition, battery health (smartphones), and the per-category bargain rate. Final strategy descriptions and the listing copy are generated by an LLM API; if the LLM is unreachable, a deterministic template fallback kicks in.

## Quick start

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Configure credentials (kept OUTSIDE the project tree)
mkdir -p ~/.monobazar-pricing-agent
cp .env.example ~/.monobazar-pricing-agent/.env
# edit ~/.monobazar-pricing-agent/.env and fill in ANTHROPIC_API_KEY (and optionally SERPAPI_KEY)

# 3. Build FAISS indexes and train the pricing models
python scripts/build_index.py

# 4. Run the API + UI
uvicorn api.app:app --reload
# open http://localhost:8000
```

> **Credentials path.** `config/settings.py` looks for `.env` first at `~/.monobazar-pricing-agent/.env`, then in the project root, then at the path in the `MONOBAZAR_ENV_FILE` environment variable. The recommended location is the home-directory one - the project tree never contains real keys, so they cannot be committed.

## API

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/api/v1/health` | Service status; number of categories and models loaded |
| POST | `/api/v1/price` | Full pricing (Phase 1 + Phase 2) |
| POST | `/api/v1/price/clarify` | Resume after the seller answers clarifying questions |
| POST | `/api/v1/price/quick` | Fast estimate without OLX or LLM (local models only) |
| POST | `/api/v1/pre-analyze` | Phase 1 only: attributes + clarifying questions |
| POST | `/api/v1/analyze-photos` | Deep photo analysis (condition, defects, brand, model) |

`POST /api/v1/price` request body:
```json
{
  "description": "iPhone 13 Pro 256GB blue, battery 92%",
  "photos": ["<base64-jpeg>", "..."],
  "category_id": 4
}
```

If `category_id` is omitted, the category is inferred from keywords in the description.

## Training and evaluation

```bash
# Build indexes and train models for all categories
python scripts/build_index.py

# Indexes only (skip training)
python scripts/build_index.py --skip-train

# Training only (indexes already exist)
python scripts/build_index.py --skip-faiss

# Single category
python scripts/build_index.py --categories 4

# Evaluate on the test split (default: SOLD listings created after 2026-03-01)
python scripts/evaluate.py
```

Per-category metrics tracked: MAPE, MAE, median APE, coverage of the [q25, q75] interval, and average model confidence. Results are written to `data/evaluation/`.

## Tests

```bash
# Regex extractor unit tests
python tests/test_extractor.py

```

## Tech stack

- **API:** FastAPI + Uvicorn
- **ML:** LightGBM (quantile regression), scikit-learn
- **Search:** FAISS (IndexFlatIP, cosine), sentence-transformers (`intfloat/multilingual-e5-large`)
- **Scraping:** OLX official API + DDGS fallback
- **LLM:** Anthropic API (vision for photo analysis, text for explanations and listing copy)
- **UI:** vanilla HTML/CSS/JS (no frameworks), mobile-first

## Security

- `.env` is in `.gitignore` and is **not** kept in the project tree by default.
- `.env.example` ships only placeholders.
- The recommended credentials location is `~/.monobazar-pricing-agent/.env` - outside the repo entirely, so it cannot be accidentally staged.
- If you ever paste a real key into the project tree, rotate it immediately at https://console.anthropic.com/settings/keys.
